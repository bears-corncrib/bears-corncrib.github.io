Birth of my Blog, Pelican based

Bear and corncobs

Just realize that lifelong learning is not only about the habit of curious for new things, but also need a good way to memerize knowledge and skills. Not smooth like a brand new car any more, my brain keep dumping something, which make me frustrated when I tried to do something that I had down long time ago, I have to do it like from fresh. What a clumsy bear I am.

Notes are helpful. I have been using Evernote, youdao note and onenote. They are all good to work like draft material repositories. When I read something and just want to keep them, in case I will need to remind sometime. The problem is that I can't share what I have practiced, emm, I have not been a developer since the start of my career. Let me see if I will have anything valuable to share.

As always, before I start using any new tools, like APP or software, especially for those I could use long time. Side by side comparison helps me to make decision. Hosting my blog, I had considered the following options.

CSDN, cnblog, segementfault, etc.

These are polular websites hosting technical blogs. They are all very handy, but I have zero control over the views of my blog, I don't like those Ads. Even they are strong in business, I still want my orignal texts stay safly in my personal cloud.

Building my own blog using wordpress

Hosting wordpress is on another extreme side. I will have full control, while I will have to spend time on managing website itself. I can learn a lot about website, about http servers, database, renting VPC, renting domain names, etc, which are not currently relevant to my daily work. So these knowledge doesn't improve my career. I would leave them in low prioriy in my "to learn" list.

Github pages, finally attracted me.

Basically, I still want the posts to be in simple format that can easier read even if from offline. Writing in markdown format which almost the same as text, satisfies my thought. I don't have to worry about transform too much when I may move my blog. Static web pages are good for simple personaly web, no Ads, tidy and focus. People are so smart using scripts to compile markdown into HTML. It is not necessary to spend time working on formats and decorite my web. There are a few scripts can be used. Here just put down some very basic ideas from my search and compare.

  • Hexo This is writen by Note.js. It is known as good performance compiling files and easy to install. There are a lot of beautiful themes which attracts people.
  • jekyll It is writen by Ruby. Performance is not good, said in some of the posts online, but it is still very polular. People don't always emphasis performance as the top 1 priority, I guess.
  • Pelican Writen by Python, which I knows better than Ruby or Node.js. This is the reason why I pick this one. There could be one day I need to modified something for my own needs.

There are more static website generators, check them out from this link https://www.staticgen.com

Building up from scratch

Build Python enviroment

It must be verbose if I introduce Python. Here in this step, just make sure you have your own Python and vertual enviroment. I am using ANACONDA.

Get Pelican and quickly build your project, which is home of your local website.

conda activate pelican Here 'pelican' is the name of my vertual enviroment.

Easly get module pelican.

pip install pelican

Alternatively, it can also be installed from source.

git clone https://github.com/getpelican/pelican
cd path-to-Pelican-source
python setup.py install

'Markdown' is strongly proposed. I don't see any use case that people host blog in github but not using Markdown as the orignal text of postes.

pip install Markdown

Then, choose a folder where you want all your project files to be stored. You can put it in you $HOME path or any sync-up folder using any cloud APP. For me, I just simply use a path in my icloud. Here below let me use $MyBlog as an alias of my own path.

cd $MyBlog

Initialize the project. 

pelican-quickstart

It generates some folers and configuration files. We are using them later.

$MyBlog/
├── content
│   └── (pages)
├── output
├── Makefile
├── pelicanconf.py       # Main settings file
└── publishconf.py       # Settings to use when ready to publish

sign up for a new account from github, and activate github page

No difficulty I can imagine when signing up a new accout. Just make sure you have account there.

If you care about the FQDN/domain name, here is my understanding. With github, you username will be used as the prefix of the FQDN, like USERNAME.github.io. I would remind that to pick a good name that you like if you don't want to spend extra effort to apply for a fancy FQDN from provider, such as https://www.name.com or https://www.godaddy.com .

There is another way to define your FQDN if your USERNAME is not what you want to use, organization. Just create a new orgnization, and use ORGANIZATION.github.io as your FQDN.

By either way above, github page need to be activated. You need to create a new reposiborty, named as name USERNAME.github.io or ORGANIZATION.github.io. It is a must, refer to Github Page. For new git user, I would propose to check "Initialize this repository with a README" when you creat your repository, otherwise you will see error next when your run git clone to build your locate repository.

After Signing up, change path to $HOMEofBLOG/output and then clone your own remote repository. 

git clone https://github.com/bears-corncrib/bears-corncrib.github.io.git

It init an local repository in output and have a full copy of your remote repository.

It can also be done step by step as following, but I would not propose you to do so unless you knows git some form already. Ignor the folowing steps and jump to next chapter.

Adding another repository, optional, if you are in a country or area that has limited bandwitch to github

For my case, I picked Gitee. Make sure you can read Chinese to use this website.

Sign up for an account in Gitee, and similarly start your gitee page. It is different from github. In gitee, you don't have to name your repository like name USERNAME.github.io or ORGANIZATION.github.io to activate page. Just name your repository withou gitee.io, then your page will be accessible from YOURREPOSITORY.gitee.io. Mine is bears-corncrib.gitee.io.

Adding another remote repository. In your local repository folder cd $HOMEofBLOG/output/YOURLOCALREPOSITORY

git remote set-url origin --add https://gitee.com/bears-corncrib/bears-corncrib.git

Check what has been changed in config.

cat .git/config 
[core]
    repositoryformatversion = 0
    filemode = true
    bare = false
    logallrefupdates = true
    ignorecase = true
    precomposeunicode = true
[remote "origin"]

    fetch = +refs/heads/*:refs/remotes/origin/*
    url = https://gitee.com/bears-corncrib/bears-corncrib.git

You can use git remote -v to check current remote repository.

git remote -v

origin  https://github.com/bears-corncrib/bears-corncrib.github.io.git (push)
origin  https://gitee.com/bears-corncrib/bears-corncrib.git (push)

Now you have both local project path and remote repositories.

Confiure your Pelican, make output folder to local repository of git

Edit Makefile to let pelican knows that which folder to output.

If your project folder path contains space, I would propose you to give a obsolute path, otherwise Pelican will compile your postes with failure. In your obsolute path, just add an escpate sign \ for all the spaces. Mine is blow as an example.

#BASEDIR=$(CURDIR)
BASEDIR=~/Library/Mobile\ Documents/com~apple~CloudDocs/MyBlog/pelicanBlog
INPUTDIR=$(BASEDIR)/content
OUTPUTDIR=$(BASEDIR)/output/bears-corncrib.github.io #Here point to your local repository.
CONFFILE=$(BASEDIR)/pelicanconf.py
PUBLISHCONF=$(BASEDIR)/publishconf.py

If you are not so good with Pelican's default theme, just download themes into a subfolder. My is $MyBlog/pelican-themes, and then give the path in pelicanconf.py. THEME = 'pelican-themes/THEMEofYOURS' There are some other parameters in pelicanconf.py, you can easily define and try.

Compose your first post

In content, make a new markdown file, extension .md. Put the following meta data in the file on the top.

    Title: My super title
    Date: 2010-12-03 10:20
    Modified: 2010-12-05 19:30
    Category: Python
    Tags: pelican, publishing
    Slug: my-super-post
    Authors: Alexis Metaireau, Conan Doyle
    Summary: Short version for index and feeds

    This is the content of my super blog post.

To understand how you should use the meta data, here is the example from this page.

    Title: Birth of my Blog, Pelican based

    Tags: github, gitee, Pelican, Blog, Python
    Slug: Birth of my Blog
    Authors: Tao

Compile your first blog

Once you have finish composing your poste in .md files. You are ready to update you blog. Go to the home of you Pelican project folder, like cd $HOMEofBLOG

Compile your post from markdown into html.

make html

You can review your page locally.

make devserver

Then, browse you blog from 0.0.0.0:8000 or localhost:8000

Warnning: Don't use make publish or make github, they clear out your .git folder from local reposibory. They are used to publish your page in another branch, gh-pages, and need another module "ghp-import", which I don't use.

publish to both of your remote repositories.

I am not using Pelican to publish blog pages to remote repositoriy, but directly using git. I would propose you to understand some of the basic use of git for this part.

Change working direcytory to cd $HOMEofBLOG/output/YOURLOCALREPOSITORY, like mine cd $MyBlog/output/bears-corncrib.github.io

git add . #This will make sure all your new files will be added to track by stage of local respository
git commit -m "Your comments" #This will submit your stage to local repository
git push -u origin master  #Push local repository to reomote repositories, both github and gitee, which defined above.

Note. Gitee.com doesn't automatically refresh you blog the same as github. You need to login your gitee account and refresh gitee page "Services->Gitee pages->Update"

Then you will be able to see your page in browser. Mine are https://bears-corncrib.github.io and https://bears-corncrib.gitee.io
