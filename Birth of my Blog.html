Birth of my Blog, Pelican based

Why I want to build a blog

Needs oriented

Different funtions from notes APP, such as Onenote or Evernote

What Candidates of bloging techniques can be chosen

CSDN, cnblog

wordpress

github pages, finally attracted me.

Basically, I still want the posts to be in simple format that can easier read even if from offline. Writing in markdown format which almost the same as text, satisfies my thought. I don't have to worry about transform too much when I may move my blog. Static web pages are good for simple personaly web, no Ads, tidy and focus. People are so smart using scripts to compile markdown into HTML. I don't have to spend time working on formats and decorite my web. There are a few scripts can be used. Here just put down some very basic ideas from my search and compare.

Hexo

This is writen by Note.js. It is known as good performance compiling files and easy to install. There are a lot of beariful themes which attracts people.

jekyll

It is writen by Ruby. Performance is not good, said in some of the posts online, but it is still very polular for some reason I don't know. People don't always emphasis performance as the top 1 priority.

Pelican

Writen by Python, which I knows better than Ruby or Node.js. This is the reason why I pick this one. There could be one day I need to modified something for my own needs. Pelican Doc,

There are more static website generators, check them out from this link

https://www.staticgen.com

How I build up my blog from scratch

  1. Build an enviroment (optional)
  2. Get Pelican and quickly build your project, which is home of your local website.
  3. sign up for a new account from github, and build activate github page

After Signing up, change path to output and git clone https://github.com/bears-corncrib/bears-corncrib.github.io.git. It init an local repository in output and have a full copy of your remote repository.

`git remote -v

origin https://github.com/bears-corncrib/bears-corncrib.github.io.git (fetch) origin https://github.com/bears-corncrib/bears-corncrib.github.io.git (push)`

  1. Adding another repository, if you are in a country of area that has limited bandwitch to github

  1. Make output folder to be local repository of git Edit Makefile BASEDIR=$(CURDIR) OUTPUTDIR

make html make devserver You can review you blog from 0.0.0.0:8000 Warnning: Don't use make publish or make github, it clear out your local reposibory.

git add . This will make sure all your new files will be added to track by stage of local respository git commit -m "Your comments" This will submit your stage to local repository git push -u origin master Push local repository to reomote repositories, which defined above.
