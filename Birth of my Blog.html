Birth of my Blog, Pelican based

Bear and corncobs

Just realize that lifelong learning is not only about the habit of curious for new things, but also need a good way to memerize knowledge and stills. Not smooth like a brand new car any more, my brain keep dumping something, which make me frustrated when I tried to do something that I had down long time ago, I have to do it like from fresh. What a clumsy bear I am.

Notes are helpful. I have been using Evernote, youdao note and onenote. They are all good to work like draft material repositories. Wenn I read something and just want to keep them in case I need to review. The problem is that I can't share what I have practiced, emm, I am not a developer. Let me see if I will have anything valuable to share.

As always, before I start using any new APP or software, especially for those I could use long time. Side by side comparison helps me to make decision. Hosting my blog, I had considered the following options.

  • CSDN, cnblog, segementfault, etc. These are polular websites hosting technical blogs. They are all very handy, but I have zero control over the views of my blog, I don't like those Ads. Even they are strong in business, I still want my orignal texts stay safly in my personal clouds.

  • Building my own blog using wordpress Hosting wordpress is on another extreme side. I have full control, while I will have to spend time on managing website. I can learn a lot, about http servers, database, renting VPC, renting domain names, etc, which are not currently relevant to my daily work. So these knowledge doesn't improve my career. I would leave them in low prioriy in my "to learn" list.

  • github pages, finally attracted me. Basically, I still want the posts to be in simple format that can easier read even if from offline. Writing in markdown format which almost the same as text, satisfies my thought. I don't have to worry about transform too much when I may move my blog. Static web pages are good for simple personaly web, no Ads, tidy and focus. People are so smart using scripts to compile markdown into HTML. I don't have to spend time working on formats and decorite my web. There are a few scripts can be used. Here just put down some very basic ideas from my search and compare.

Hexo

This is writen by Note.js. It is known as good performance compiling files and easy to install. There are a lot of beariful themes which attracts people.

jekyll

It is writen by Ruby. Performance is not good, said in some of the posts online, but it is still very polular for some reason I don't know. People don't always emphasis performance as the top 1 priority, I guess.

Pelican

Writen by Python, which I knows better than Ruby or Node.js. This is the reason why I pick this one. There could be one day I need to modified something for my own needs. Pelican Doc,

There are more static website generators, check them out from this link

https://www.staticgen.com

How I built up my blog from scratch

Build Python enviroment

It must be verbose if I introduce python. Here in this step, just make sure you have your own python and vertual enviroment. I am using ANACONDA.

Get Pelican and quickly build your project, which is home of your local website.

conda activate pelican Here 'pelican' is the name of my vertual enviroment.

Easly get module pelican.

pip install pelican

Alternatively, it can also be installed from source.

git clone https://github.com/getpelican/pelican
cd path-to-Pelican-source
python setup.py install

'Markdown' is strongly proposed. I don't see any use case that people host blog in github but not using Markdown as the orignal text of postes.

pip install Markdown

Then, choose a folder where you want all your project files to be stored. You can put it in you $HOME path or any sync-up folder using any cloud APP. For me, I just simply use a path in my icloud. Here below let me use $MyBlog as an alias of my own path.

cd $MyBlog Initialize the project. It generates some folers and configuration files. We will be using them later.

sign up for a new account from github, and activate github page

After Signing up, change path to output and git clone https://github.com/bears-corncrib/bears-corncrib.github.io.git. It init an local repository in output and have a full copy of your remote repository. It can be down by step by step: make a folder name what ever you like in output and change into it, mine is $HOMEofBLOG/output/bears-corncrib.github.io git init git remote add origin https://github.com/bears-corncrib/bears-corncrib.github.io.git You can use git remode -v to check current remote repository.

`git remote -v

origin  https://github.com/bears-corncrib/bears-corncrib.github.io.git (fetch)
origin  https://github.com/bears-corncrib/bears-corncrib.github.io.git (push)`

Adding another repository, if you are in a country of area that has limited bandwitch to github

For my case, I picked gitee.

git remote set-url origin --add https://gitee.com/bears-corncrib/bears-corncrib.git

`git remote -v
origin  https://github.com/bears-corncrib/bears-corncrib.github.io.git (fetch)
origin  https://github.com/bears-corncrib/bears-corncrib.github.io.git (push)
origin  https://gitee.com/bears-corncrib/bears-corncrib.git (push)`

Add the changes you did above can check in $MyBlog/.git/config

cat .git/config 
[core]
    repositoryformatversion = 0
    filemode = true
    bare = false
    logallrefupdates = true
    ignorecase = true
    precomposeunicode = true
[remote "origin"]
    url = https://github.com/bears-corncrib/bears-corncrib.github.io.git
    fetch = +refs/heads/*:refs/remotes/origin/*
    url = https://gitee.com/bears-corncrib/bears-corncrib.git
[branch "master"]
    remote = origin
    merge = refs/heads/master

Make output folder to be local repository of git

Now you have both local project path and remote repositories. Edit Makefile to let pelican knows that which folder to output OUTPUTDIR=$(BASEDIR)/output/bears-corncrib.github.io

Compose your first post

In content, make a new .md file. Put the following meta data in the file on the top.

    Title: My super title
    Date: 2010-12-03 10:20
    Modified: 2010-12-05 19:30
    Category: Python
    Tags: pelican, publishing
    Slug: my-super-post
    Authors: Alexis Metaireau, Conan Doyle
    Summary: Short version for index and feeds

    This is the content of my super blog post.

Compile your first blog

make html make devserver You can review you blog from 0.0.0.0:8000 Warnning: Don't use make publish or make github, it clear out your local reposibory. * publish to both of your remote repositories.

git add . This will make sure all your new files will be added to track by stage of local respository git commit -m "Your comments" This will submit your stage to local repository git push -u origin master Push local repository to reomote repositories, which defined above.

Note. Gitee.com doesn't automatically refresh you blog the same as github. You need to login your gitee account and refressh gitee page "Services-Gitee pages-Update"
